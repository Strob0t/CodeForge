When writing documentation:
1. Keep docs close to the code they describe.
2. Document the "why" — the code shows the "what".
3. Use examples for APIs and complex functions.
4. Keep READMEs focused: what it does, how to use it, how to contribute.
5. Update docs when you change the code they describe.
6. Avoid stale comments — remove comments that no longer match the code.
